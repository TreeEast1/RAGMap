# FAQ

## 1. 这个仓库是论文列表吗？

不是。  
论文会逐步补，但它们服务于“方法演进地图”，不是仓库的唯一目标。

## 2. 我是初学者，应该从哪里开始？

从 [docs/01-what-is-rag.md](./docs/01-what-is-rag.md) 和 [docs/02-basic-rag.md](./docs/02-basic-rag.md) 开始，再读 [docs/03-why-basic-rag-is-not-enough.md](./docs/03-why-basic-rag-is-not-enough.md)。

## 3. 为什么要单独讨论 GraphRAG、MemoryRAG、Thinking RAG？

因为它们代表的不是“不同名词”，而是不同的问题意识：

- GraphRAG 更关注结构化组织与全局关系；
- MemoryRAG 更关注长期知识组织；
- Thinking RAG 更关注检索与推理协同。

## 4. CognitiveRAG 在这个仓库中的定位是什么？

它不是被单独宣传的孤立方法，而是作为 RAG 演进中的一个重要方向来讨论：当检索、记忆、重排和推理被放进更统一的认知框架时，会带来什么新可能，也会带来什么复杂度。

## 5. 这个项目未来会做成网站吗？

有可能，但第一阶段重点是把仓库内容结构做稳，保证 GitHub 首页和文档层次足够清晰。
